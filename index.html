PROBLEM & HYPOTHESIS

痛點與假說

⬡ 教學現場的真實困難

六年級「雞兔同籠」單元迷思多——腳數弄反、不懂 ±2 規律、假設法答反，
而且老師看不見全班「當下」卡在哪裡。

⬡ 我們的假說

如果用 AI Agent 把討論直接變成可上線的遊戲與後臺，
就能即時看見卡關、老師專注教學專業。

DAY 1 · 想法彙整與配對

錄音 → 逐字稿 → 問題定義單

  • 1

    🎙️ 用 iPhone 錄音機錄下整場討論——大家盡情丟想法，不必有人分心做紀錄。

  • 2

    🤖 Agent 消化逐字稿，自動把每個人的想法配對、填進 MVP 問題定義單。

  • 3

    👀 人做 double check：檢查 Agent 有沒有填對，給回饋修正。

  • 4

    🎯 收斂出主題：六年級數學「雞兔同籠」遊戲化，定義好對象、限制與小目標。

DAY 2 · 從定義到上線

教冊 PDF 進去，遊戲網站出來

  • 1

    針對問題單內容再次深化討論（一樣全程錄音）。

  • 2

    丟入「怎樣解題」教冊 PDF：「做一個符合我們問題定義單的東西」——Agent 產出四關卡遊戲。

  • 3

    一句話「直接上架」→ GitHub Pages 部署完成，學生開網址就能玩。

  • 4

    先討論再動手：想看學生卡在哪 → 決定「課堂即時看、班級＋座號、每關就好」→ Agent 加上老師端即時儀表板。

  • 5

    實測回饋：第 4 關數字太大、加獎狀功能——截圖回饋，Agent 修正。

COMPARISON

組員實測：傳統方法 vs Agent 輔助

⬡ 傳統方法
  • ✦要有人分心做會議紀錄
  • ✦表單、簡報、文書自己熬夜做
  • ✦想做互動教材要會寫程式或找廠商
  • ✦重複性流程每次都重走一遍
VS
⬡ Agent 輔助
  • ✦錄音即紀錄，想法自動進表單
  • ✦定義夠清楚，成品直接生出來
  • ✦一句話部署上線＋串資料庫
  • ✦老師專注：提問、判斷、教學專業

※ 人沒有消失——每一步都要 double check，教學專業才是方向盤。

HUMAN × AI

人機分工：誰做什麼？

🧑‍🏫 老師（方向盤）
  • ▸提出真實問題、定義清楚需求
  • ▸教學專業判斷：數字難度、關卡順序、迷思點
  • ▸double check 每個產出、截圖給回饋
🤖 Agent（引擎）
  • ▸逐字稿消化、表單填寫、文書瑣事
  • ▸寫程式、做遊戲、串資料庫
  • ▸部署上架、除錯、教你它做了什麼

🧒 學生呢？AI 不代替學生思考——遊戲只給診斷式回饋，觀察、列表、假設、驗算都是學生自己來。
